# 1. Постановка і аналіз задачі

**Суть проєкту**: розробка карткової стратегічної гри у жанрі колекційних карткових ігор (ККГ). Проєкт реалізується на Java з використанням графічної бібліотеки JavaFX.

**Мета гри**: два гравці по черзі розігрують картки зі своїх колод, намагаючись знизити кількість очок здоров'я суперника з 20 до нуля. Кожен хід гравці набирають картки з колоди та розігрують їх за ману - ресурс, який надають картки Земель, та оновлюються на початку кожного ходу.

**Основні функціональні вимоги**:

- *Ігровий процес 1 на 1*: повноцінна дуель між двома гравцями зі спільним ігровим полем, почерговою структурою ходів та системою перемоги/поразки;

- *Система карток*: гра включає як мінімум три основних типи карток:
    - **Землі** - надають ману для розіграшу інших карток; виставляються на поле та оновлюються щоразу на початку ходу свого власника;
    - **Істоти** - залишаються на полі після розіграшу, атакують суперника або його істот та можуть мати особливі перки;
    - **Закляття** - одноразові картки миттєвої дії, що створюють несподівані або потужні ефекти;

- *Система перків істот*: окремі істоти мають унікальні здібності, що суттєво впливають на перебіг бою:
    - **Політ** - істоту не можуть заблокувати наземні істоти, що дозволяє завдавати шкоди безпосередньо гравцю;
    - **Ланцюг життя** - власник отримує очки здоров'я, рівні завданій цією істотою шкоді;
    - **Пробивний удар** - якщо показник атаки, вищий ніж показник здоров'я блокуючої істоти, то залишок шкоди не втрачаєтся, а йде далі(у істоту, або одразу у гравця);
    - **Досяжність** - істота з цим перком досягає до літаючих створінь, що дає їй змогу їх блокувати

- *Бойова система*: програма повинна коректно обробляти оголошення атакуючих та вибір блокуючих істот, розраховувати шкоду та визначати виживання або загибель кожної з них;

- *Ігровий цикл*: потребує реалізації структурованої почерговості ходів (набір карток, поповнення мани, головна фаза, бойова фаза), обробки станів гри та коректного завершення партії за умов перемоги або поразки.

## Розподіл ролей всередині команди

Наша команда складається з 4 осіб. Для оптимізації процесів та паралельної роботи завдання було розділено на чотири ключових архітектурних блоки:

- **Архітектура проєкту, класи карток та стани гри**. Відповідальний: **Артем**.
    - Проєктування загальної архітектури системи;
    - Розробка ієрархії класів карток (Земля, Істота, Закляття) та їх властивостей;
    - Реалізація класів гравців та управління їх станом (здоров'я, мана, рука, колода, поле);
    - Опис і реалізація ефектів карток та механіки перків.

- **Ігровий двигун та логіка бою**. Відповідальний: **Олександр**.
    - Реалізація повного циклу ходу (всі фази від початку до кінця);
    - Алгоритм розрахунку бою між істотами та нанесення шкоди гравцям;
    - Застосування ефектів заклять та обробка ланцюгів дій.

- **Графічний інтерфейс (JavaFX)**. Відповідальний: **Гордій**.
    - Розробка та відображення ігрового поля;
    - Візуалізація руки гравця, виставлених на поле карток;
    - Відображення поточного рівня здоров'я та мани обох гравців;
    - Реалізація візуального вибору атакуючих та блокуючих істот.

- **AI, дані та тестування**. Відповідальний: **Артур**.
    - Розробка штучного інтелекту для гри проти комп'ютера;
    - Реалізація репозиторію карток та завантаження їх із JSON-файлів;
    - Написання JUnit5 тестів для модулів `CombatResolver` та `EffectExecutor`;
    - Балансування ігрових параметрів карток.

## Опис технічних засобів розробки

Для реалізації проєкту обрано сучасний та стабільний стек технологій:

- **Мова програмування**: Java (версія 17 або вище);

- **Графічна бібліотека**: JavaFX - основний інструмент для побудови візуальної частини гри. У проєкті застосовуються:
    - *Scene* та *Stage* - для менеджменту вікон програми (головне меню, ігрове поле, екран результатів);
    - *FXML* - для декларативного опису макету ігрового інтерфейсу;
    - *CSS-стилізація* - для оформлення карток, кнопок та ігрового поля;
    - *AnimationTimer* / *Timeline* - для відтворення анімацій розіграшу карток та бойових ефектів;

- **Формат даних**: JSON - для зберігання репозиторію карток (назви, характеристики, тип, перки, ефекти);

- **Фреймворк тестування**: JUnit 5 - для написання модульних тестів бойової логіки та ефектів;

- **Середовище розробки (IDE)**: IntelliJ IDEA;

- **Система автоматизації збирання проєктів**: Maven;

- **Контроль версій**: Git та платформа GitHub.
